1//! # ratatui_testlib
2//!
3//! A Rust library for integration testing of terminal user interface (TUI) applications.
4//!
5//! ## Overview
6//!
7//! `ratatui_testlib` provides a comprehensive testing framework for TUI applications built with
8//! libraries like Ratatui, with first-class support for:
9//!
10//! - **PTY-based testing**: Real terminal emulation with pseudo-terminal support
11//! - **Sixel graphics testing**: Position verification and bounds checking for Sixel images
12//! - **Bevy ECS integration**: Test Bevy-based TUI applications with `bevy_ratatui`
13//! - **Async runtime support**: Full Tokio async/await support
14//! - **Snapshot testing**: Integration with `insta` for visual regression testing
15//! - **Headless CI/CD**: Works without X11/Wayland for GitHub Actions
16//!
17//! ## Quick Start
18//!
19//! ### PTY-Based Testing (Full TUI Applications)
20//!
21//! ```rust,no_run
22//! use portable_pty::CommandBuilder;
23//! use ratatui_testlib::{Result, TuiTestHarness};
24//!
25//! #[test]
26//! fn test_my_tui_app() -> Result<()> {
27//!     // Create a test harness with 80x24 terminal
28//!     let mut harness = TuiTestHarness::new(80, 24)?;
29//!
30//!     // Spawn your TUI application
31//!     let mut cmd = CommandBuilder::new("./my-tui-app");
32//!     harness.spawn(cmd)?;
33//!
34//!     // Wait for initial render
35//!     harness.wait_for(|state| state.contents().contains("Welcome"))?;
36//!
37//!     // Send keyboard input
38//!     harness.send_text("hello")?;
39//!
40//!     // Capture screen state
41//!     let contents = harness.screen_contents();
42//!     assert!(contents.contains("hello"));
43//!
44//!     Ok(())
45//! }
46//! ```
47//!
48//! ### Stream-Based Parsing (Headless/Oracle Mode)
49//!
50//! For testing terminal emulators or parsing raw escape sequences without PTY overhead:
51//!
52//! ```rust
53//! use ratatui_testlib::ScreenState;
54//!
55//! #[test]
56//! fn test_ansi_sequence_parsing() {
57//!     // Create parser without PTY
58//!     let mut screen = ScreenState::new(80, 24);
59//!
60//!     // Feed raw byte sequence
61//!     let input = b"\x1b[31mHello\x1b[0m";
62//!     screen.feed(input);
63//!
64//!     // Verify parsed state
65//!     assert!(screen.contains("Hello"));
66//!     assert_eq!(screen.get_cell(0, 0).unwrap().fg, Some(1)); // Red
67//!     assert_eq!(screen.cursor_position(), (0, 5));
68//! }
69//! ```
70//!
71//! ## Testing Sixel Graphics
72//!
73//! With the `sixel` feature enabled, you can verify Sixel graphics positioning:
74//!
75//! ```rust,no_run
76//! # #[cfg(feature = "sixel")]
77//! # {
78//! use ratatui_testlib::TuiTestHarness;
79//!
80//! # fn test_sixel() -> ratatui_testlib::Result<()> {
81//! let mut harness = TuiTestHarness::new(80, 24)?;
82//! // ... spawn your app and trigger Sixel rendering ...
83//!
84//! // Verify all Sixel graphics are within the preview area
85//! let preview_area = (5, 5, 30, 15); // row, col, width, height
86//! let sixel_regions = harness.state().sixel_regions();
87//! for region in sixel_regions {
88//!     assert!(region.start_row >= preview_area.0);
89//!     assert!(region.start_col >= preview_area.1);
90//! }
91//! # Ok(())
92//! # }
93//! # }
94//! ```
95//!
96//! ## Testing with Shared Memory State
97//!
98//! With the `shared-state` feature, you can access memory-mapped shared state:
99//!
100//! ```rust,no_run
101//! # #[cfg(all(feature = "bevy", feature = "shared-state"))]
102//! # {
103//! use ratatui_testlib::{
104//!     shared_state::{MemoryMappedState, SharedStateAccess},
105//!     BevyTuiTestHarness,
106//! };
107//! use serde::{Deserialize, Serialize};
108//!
109//! #[derive(Debug, Clone, Serialize, Deserialize)]
110//! struct AppState {
111//!     frame_count: u32,
112//!     status: String,
113//! }
114//!
115//! # fn test() -> ratatui_testlib::Result<()> {
116//! let harness = BevyTuiTestHarness::new()?.with_shared_state("/tmp/app_state.mmap")?;
117//!
118//! // Access shared state for assertions
119//! if let Some(path) = harness.shared_state_path() {
120//!     let state = MemoryMappedState::<AppState>::open(path)?;
121//!     let app_state = state.read()?;
122//!     assert!(app_state.frame_count > 0);
123//! }
124//! # Ok(())
125//! # }
126//! # }
127//! ```
128//!
129//! ## Feature Flags
130//!
131//! - `async-tokio`: Enable Tokio async runtime support
132//! - `bevy`: Enable Bevy ECS integration
133//! - `bevy-ratatui`: Enable bevy_ratatui plugin support
134//! - `ratatui-helpers`: Enable Ratatui-specific test helpers
135//! - `sixel`: Enable Sixel graphics position tracking and testing
136//! - `snapshot-insta`: Enable snapshot testing with `insta`
137//! - `headless`: Enable headless mode for CI/CD (no display server required)
138//! - `shared-state`: Enable memory-mapped shared state access for testing
139//! - `mvp`: Enable all MVP features (recommended for dgx-pixels)
140//!
141//! ### Headless Mode for CI/CD
142//!
143//! The `headless` feature flag configures the library to run without any display
144//! server dependencies, making it ideal for CI/CD environments:
145//!
146//! ```bash
147//! # Run tests in CI without X11/Wayland
148//! cargo test --features bevy,headless
149//!
150//! # Works in Docker containers
151//! docker run --rm rust:latest cargo test --features bevy,headless
152//! ```
153//!
154//! In headless mode, the Bevy integration uses `MinimalPlugins` instead of
155//! `DefaultPlugins`, eliminating all graphics and windowing dependencies.
156//!
157//! ## Architecture
158//!
159//! The library is organized into several layers:
160//!
161//! 1. **PTY Management** (`pty`): Pseudo-terminal creation and lifecycle
162//! 2. **Terminal Emulation** (`screen`): VT100 parsing and screen state
163//! 3. **Test Harness** (`harness`): High-level testing API
164//! 4. **Sixel Support** (`sixel`): Graphics protocol testing (MVP requirement)
165//! 5. **Bevy Integration** (`bevy`): ECS testing support (MVP requirement)
166//!
167//! ## Primary Use Case: dgx-pixels
168//!
169//! This library was built to support comprehensive integration testing for the
170//! [dgx-pixels](https://github.com/raibid-labs/dgx-pixels) project, with focus on:
171//!
172//! - Sixel graphics positioning within designated preview areas
173//! - Sixel clearing on screen transitions
174//! - Bevy ECS entity and component testing
175//! - Text input and cursor position verification
176//! - Async Tokio runtime compatibility
